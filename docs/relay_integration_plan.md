# 📡 WebTerm 安卓客户端 - 多设备中转服务集成方案

本方案旨在为 `WebTerm Android 客户端` 集成对多设备中转服务器（`relay-server`）的原生支持，打通安卓端在大厅动态拉取、切换、管理多台电脑设备（`pc-agent`）的闭环，并在交互体验上进行深度优化。

---

## 📌 整体设计方案

### 1. 顶部菜单集成（入口）
*   **动作项**：在大厅主界面的右上角“三个点”按钮下拉菜单中，追加 **“🔗 中转服务”** 菜单项。
*   **对话框**：点击后弹出对话框，由用户配置中转服务的相关凭据：
    *   中转服务器 URL（例如：`http://100.121.115.14:9000`）
    *   用户名（Username）
    *   密码（Password）
*   **凭证管理**：支持“登录并保存配置”和“退出登录”两个动作。

### 2. 状态融合渲染（副标题替换）
为了保持大厅界面的简洁，不占用垂直空间，我们将**原本的副标题“多端会话聚合大厅”作为状态提醒区**：

| 中转连接状态 | 副标题显示文本 | 文本颜色 (RGB) | 说明 |
| :--- | :--- | :--- | :--- |
| **未配置 / 已退出** | `⚠️ 中转服务未连接` | `Color.rgb(245, 158, 11)` (橙黄) | 用户尚未配置中转服务器，或点击了断开连接。 |
| **连接中** | `⏳ 正在连接中转服务...` | `Color.rgb(245, 158, 11)` (橙黄) | 正在与中转服务器建立控制通道。 |
| **连接失败** | `🚨 中转服务连接失败` | `Color.rgb(239, 68, 68)` (红) | 握手失败、鉴权失败或网络超时。 |
| **已成功连接** | `🟢 已连接中转服务` | `Color.rgb(16, 185, 129)` (绿) | 成功取得在线设备，大厅准备加载数据。 |

### 3. 多设备动态聚合大厅
*   **凭据持久化**：使用专有的 `RelayConfigStore` 来存取中转配置，与普通直连电脑配置分离。
*   **长连接动态加载**：
    *   大厅载入时，客户端通过 WebSocket 自动与中转服务的控制通道（`/ws/sessions`）建立长连接。
    *   成功握手后，中转服务器会自动向客户端推送当前名下的所有在线受控设备列表（`devices`）。
    *   客户端为每个在线受控设备在 UI 大厅上渲染为一个“受控设备组”。
*   **中转图标前缀**：中转设备的标题前自动拼接中转的特殊图标 `🔗`（如：`🔗 办公室的电脑`），以便用户区分普通直连电脑和中转电脑。

## 🛠️ 底层网络与协议适配 (API Layer)

由于中转服务器支持多台设备，我们需要在数据请求与 WebSocket 连接中带上设备标识符，否则请求会报 `503 PC Agent offline` 错误。

### 1. HTTP 适配：自动填充 `x-device-id` 头部
*   在 `WebTermApi.java` 中，针对所有对中转服务器的 API 调用（如 `fetchSessions`、`createSession`、`deleteSession`），如果在 `ServerConfig` 中检测到这是一个中转服务下的特定设备，就在 HTTP 请求头中添加 `x-device-id`，值为该设备的 `deviceId`。

### 2. WebSocket 适配：全局会话 ID 拼接
*   连接中转终端的 WebSocket 数据通道时，其 `sessionId` 格式在底层为 `deviceId:localSessionId`，使得中转服务器能够解析前缀，进行链路转发。
